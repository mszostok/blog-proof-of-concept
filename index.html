Localhost Setup


1. Database

First we need to provide postgresql server, you can use for example pgAdmin

Then you need to create database with name ‘blog’, owner ‘postgres’ and password ‘123456’ or if you want you can change this settings in application.properties file with your own data (different db name, owner or password ).

For now I don’t support environment variables for database connection.

When you have your database created, you need to execute DDL (link) and SQL (link) script to setup proper tables and sample data.

2. Environment variable

Due to we can upload some images when we creating new notes, you need to provide some directory path where you want to store them. It cannot be in resources application folder because when redeploy application we lose also our data, so please create environment variable named UPLOAD_LOCATION, for example

Windows

setx UPLOAD_LOCATION "D:/path/to/uploads/"

Linux

export UPLOAD_LOCATION ="/path/to/uploads/"

3. Run

When you have your database and environment variable setup, you can clone repository for example using git-clone

git clone https://github.com/mszostok/blog-proof-of-concept.git

Then import project via pom.xml to your IDE, and execute maven command

mvn clean spring-boot:run

in this case you will start application with embedded server (tomcat). If you want you can also deploy it to an existing Tomcat installation, or to the classic Java EE application server via executing

mvn clean package

which will create .war file.
